同一句 prompt
三次生成，三种风格

没有设计规范时，AI 自由发挥——颜色随机、字体混搭、圆角各异。 设计师不是在审核设计，是在修图。

"大多数时间花在统一风格上，而不是迭代功能。"
—— 设计师的真实反馈
风格漂移 经常发生 同一 prompt 产出差异明显
Button →
生成 #1 · 暖色大圆角
Submit
生成 #2 · 冷色直角
确认
生成 #3 · 绿色混合

四步工作流

01 读取规范 Codex Figma 插件
读取设计规范链接 →
02 生成描述 design-generator skill
输出 design.md →
03 内置规范 design.md 内置到
OpenDesign 系统风格 →
04 生成原型 OpenDesign 产出
符合规范的界面
!
两个坦诚的认知：
❶ 该方法只能在一定程度下规范原型风格——不会 100% 一致，但能把漂移控制在一个可接受的范围。
❷ 输出效果高度依赖输入质量：Figma 设计规范越完整、产品需求文档越清晰，生成的原型就越贴近预期。反之亦然——输入模糊，输出也模糊。
